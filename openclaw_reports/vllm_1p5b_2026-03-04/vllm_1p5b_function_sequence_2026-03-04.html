3) 数据流分层说明

  1. API层：文本 prompt + sampling params 进入。
  2. AsyncLLM层：封装成 EngineCoreRequest，进入异步队列。
  3. Core/Scheduler层：执行 prefill/decode 调度，分配/复用 KV cache。
  4. GPU Worker/Runner层：执行前向，输出 new_token_ids。
  5. OutputProcessor层：detokenize + stop 检查 + usage 统计。
  6. Serving层：组装 OpenAI JSON（choices/usage/finish_reason）并返回。

4) 性能定位优先观测点

  • core.py::step：调度节奏、等待/饥饿状态
  • gpu_model_runner.py::execute_model：prefill vs decode 算力压力
  • output_processor.py::process_outputs：流式输出节拍与结束条件
  • 结合 runtime 日志：throughput、KV cache usage、prefix cache hit rate
