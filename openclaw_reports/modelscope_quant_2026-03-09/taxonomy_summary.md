## 三、把你的 taxonomy 和 ModelScope 现实生态对齐后的结论
1. **GGUF 是压倒性第一大类**
   - 如果从平台发布形态看，GGUF 远大于所有其他单一技术路线。
   - 这说明 ModelScope 的公开分发生态里，边缘侧 / 本地部署 / 单文件封装格式非常强势。

2. **PTQ 主流路线仍然是 AWQ + GPTQ**
   - AWQ：2,000
   - GPTQ：1,887
   - HQQ：86
   - EXL2：781
   - 四者合计：4,754（13.70%）
   - 这说明“真正明确按 PTQ 算法名发布”的主流仍然集中在 AWQ / GPTQ。

3. **硬件原生标准路线正在增长，但在公开模型命名层面还不是主流**
   - 标准 FP8：1,052
   - NVFP4：207
   - MXFP4 / MXFP6：206
   - MXFP8 / MXINT8：25
   - 合计 1,490（4.30%）
   - 说明 FP8 已经形成存在感，但 MX / NVFP4 还更偏前沿与特定生态。

4. **全链路量化（SmoothQuant / W8A8）在平台可见度不高，但工程价值不低**
   - 数量只有 465（1.34%）
   - 这更像“服务部署优化路线”，不是公开模型分发最主流的命名方式。

5. **NF4 / AQLM / BitNet 等前沿或专属路线目前仍属于长尾**
   - NF4：41
   - AQLM / QuIP#：29
   - BitNet b1.58：46
   - 这符合预期：它们在研究/特定场景里重要，但在 ModelScope 平台分发层面还不是主流。

6. **最大现实问题：大量模型没有写明算法名**
   - 有 10,553 个模型只能归到“通用位宽命名”
   - 这说明如果只按 AWQ/GPTQ/NF4 这类算法名看，会明显低估真实量化生态规模

## 五、关键结论（一句话版）
如果用你这套 2026 taxonomy 去看 ModelScope：
- **平台最主流的是 GGUF 这类边缘/封装生态**
- **算法名最主流的是 AWQ + GPTQ**
- **硬件原生标准（FP8/MX/NVFP4）已出现明显增长，但仍不是平台主流发布形态**
- **真正最大的噪声/现实约束，是大量模型只写位宽，不写算法名**
